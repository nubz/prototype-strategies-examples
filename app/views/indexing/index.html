Indexing examples

Why bother with versions? Why not just keep designing forward and overwriting the old designs? Some possible reasons:

  • versions provide snapshots of each iteration
  • versions enable the journeys tested with users to be used in the future
  • versions tell the story

A system for organising prototype designs and journeys by versions. Consideration in this system has been given to the representation of change across versions.

  • indexing all of the pages in each version
  • keeping track of changes between versions
  • comparing versions of the same page across versions

The template, routing and model code for all examples can be seen on Github.
